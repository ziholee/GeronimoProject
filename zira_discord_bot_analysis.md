# Zira Discord Bot 기능 상세 분석

## 1. 개요

**Zira**는 Discord 서버에서 **Reaction Role**, 즉 이모지 반응을 통해 유저에게 역할을 자동으로 부여하거나 제거하는 데 특화된 봇이다.

Zira의 핵심 목적은 다음과 같다.

- 유저가 직접 역할을 선택하게 하기
- 서버 입장 후 규칙 동의 및 인증 처리
- 알림 구독 역할 관리
- 파트, 관심사, 게임, 지역 등 역할 분류 자동화
- 신규 유저 또는 봇에게 자동 역할 부여
- 서버 온보딩 구조 자동화

Zira는 MEE6, Carl-bot, Dyno처럼 다양한 기능을 제공하는 종합 관리 봇이라기보다는 **역할 자동화에 집중된 전문 봇**에 가깝다.

---

## 2. 핵심 기능 요약

| 기능 | 설명 |
|---|---|
| Reaction Role | 이모지 반응으로 역할을 부여하거나 제거하는 기능 |
| Normal Role | 반응 추가 시 역할 부여, 반응 제거 시 역할 제거 |
| Once Role | 한 번만 반응 가능하며, 규칙 동의나 인증에 적합 |
| Remove Role | 반응 시 특정 역할 제거 |
| Toggle Role | 여러 선택지 중 하나의 역할만 유지 |
| Autorole | 서버 입장 시 자동 역할 부여 |
| Timed Role | 일정 시간이 지난 뒤 역할 부여 |
| Message | Zira가 직접 역할 선택 메시지 생성 |
| Embed | 깔끔한 임베드 메시지 생성 |
| Suggestions | 서버 제안 시스템 운영 |
| List/Delete | 설정된 역할 반응 확인 및 삭제 |

# 3. Reaction Role 기능

Zira의 핵심 기능은 **Reaction Role**이다.

관리자가 특정 메시지에 이모지를 연결하면, 유저가 그 이모지를 눌렀을 때 자동으로 역할이 부여된다.

예시:

```txt
🤖 → AI 파트
💻 → 프론트엔드
🛠 → 백엔드
🎨 → 기획/디자인
📢 → 공지 알림
🏆 → 해커톤 알림
📚 → 스터디 알림
```

Reaction Role은 서버 내에서 유저가 직접 자신의 관심사나 소속을 선택하게 만들 수 있기 때문에, 커뮤니티 운영 효율을 크게 높여준다.

---

## 3.1 Normal Role

**Normal Role**은 가장 기본적인 역할 반응 방식이다.

유저가 이모지를 누르면 역할이 부여되고, 반응을 제거하면 역할도 제거된다.

### 사용 예시

```txt
📢 공지 알림
🏆 해커톤 알림
📚 스터디 알림
🤖 AI 파트
💻 프론트엔드
🛠 백엔드
🎨 기획/디자인
```

### 적합한 상황

- 관심사 선택
- 알림 구독
- 스터디 파트 선택
- 게임 역할 선택
- 프로젝트 분야 선택
- 플랫폼 선택
- 취미나 활동 분야 선택

### 장점

- 가장 직관적이다.
- 유저가 여러 역할을 동시에 선택할 수 있다.
- 대부분의 서버에서 가장 많이 쓰이는 방식이다.
- 알림 구독이나 관심사 분류에 적합하다.

### 주의점

Discord 자체 제한으로 한 메시지에 붙일 수 있는 반응은 보통 **최대 20개**까지다.  
역할이 많다면 메시지를 여러 개로 나누는 것이 좋다.

---

## 3.2 Once Role

**Once Role**은 한 번만 사용할 수 있는 역할 반응이다.

유저가 이모지를 누르면 역할이 부여되고, 이후 해당 반응은 제거되거나 다시 사용할 수 없도록 처리된다.

### 사용 예시

```txt
✅ 규칙에 동의합니다 → Verified 역할 부여
```

### 적합한 상황

- 서버 규칙 동의
- 인증 완료 처리
- 신규 멤버 온보딩
- 특정 채널 접근 권한 부여
- 일회성 확인 절차

### 장점

- 인증 시스템처럼 사용할 수 있다.
- 유저가 실수로 반응을 제거해도 역할이 쉽게 사라지지 않는다.
- 규칙 동의용 메시지에 적합하다.
- 신규 유저 온보딩을 단순화할 수 있다.

### 주의점

Once Role은 다른 Reaction Role 타입과 섞지 않는 것이 좋다.  
규칙 동의 메시지는 독립된 메시지로 분리하는 편이 안전하다.

---

## 3.3 Remove Role

**Remove Role**은 반응을 누르면 특정 역할을 제거하는 기능이다.

### 사용 예시

```txt
🔕 공지 알림 해제
❌ 이벤트 알림 해제
🚫 임시 역할 제거
```

### 적합한 상황

- 알림 구독 해제
- 임시 역할 제거
- 특정 상태 해제
- 기존 역할 정리
- 유저가 직접 역할을 빼야 하는 상황

### 장점

- 유저가 직접 필요 없는 역할을 제거할 수 있다.
- 관리자 개입 없이 역할 정리가 가능하다.
- 알림 구독 해제 구조를 만들 수 있다.

### 단점

- 무료 플랜에서는 개수 제한이 있을 수 있다.
- 역할 제거 구조가 복잡해지면 관리가 어려워질 수 있다.

---

## 3.4 Toggle Role

**Toggle Role**은 여러 선택지 중 하나만 유지하도록 하는 역할 방식이다.

예를 들어 색상 역할, 학년, 기수, 포지션처럼 **동시에 여러 개를 가지면 안 되는 역할**에 적합하다.

### 사용 예시 1: 색상 선택

```txt
🔴 빨간색 닉네임
🔵 파란색 닉네임
🟢 초록색 닉네임
```

### 사용 예시 2: 학년 선택

```txt
1️⃣ 1학년
2️⃣ 2학년
3️⃣ 3학년
4️⃣ 4학년
```

### 사용 예시 3: 파트 선택

```txt
🤖 AI 파트
💻 프론트엔드
🛠 백엔드
🎨 기획/디자인
```

### 적합한 상황

- 닉네임 색상 선택
- 학년 선택
- 기수 선택
- 지역 선택
- 포지션 선택
- 하나만 선택해야 하는 분류
- 중복 선택을 방지해야 하는 역할 구조

### 장점

- 중복 선택을 방지할 수 있다.
- 서버 역할 구조가 깔끔해진다.
- 유저가 하나의 대표 역할만 가지도록 만들 수 있다.

### 단점

- 무료 플랜에서는 사용 개수 제한이 있을 수 있다.
- 여러 선택지를 운영할 때 구조를 잘 설계해야 한다.

# 4. 자동 역할 기능

## 4.1 Autorole

**Autorole**은 유저나 봇이 서버에 들어왔을 때 자동으로 역할을 부여하는 기능이다.

### 사용 예시

```txt
신규 유저 → New Member 역할 자동 부여
신규 봇 → Bot 역할 자동 부여
```

### 적합한 상황

- 신규 유저에게 기본 역할 부여
- 봇에게 Bot 역할 자동 부여
- 미인증 유저 격리
- 온보딩 단계 분리
- 서버 입장 직후 기본 권한 관리

### 활용 구조 예시

```txt
서버 입장
↓
New Member 역할 자동 부여
↓
규칙 동의
↓
Verified 역할 부여
↓
일반 채널 접근 가능
```

### 장점

- 신규 유저 관리가 쉬워진다.
- 인증 전/후 권한을 분리할 수 있다.
- 봇과 일반 유저를 자동으로 구분할 수 있다.

---

## 4.2 Timed Role

**Timed Role**은 유저가 서버에 들어온 뒤 일정 시간이 지나면 역할을 부여하는 기능이다.

### 사용 예시

```txt
가입 직후 → New Member
10분 후 → Member
```

### 적합한 상황

- 신규 유저 대기 시간 부여
- 스팸 계정 방지
- 일정 시간 후 일반 멤버 전환
- 단계별 온보딩
- 신입 멤버 적응 기간 운영

### 장점

- 서버 보안과 온보딩을 동시에 관리할 수 있다.
- 즉시 모든 권한을 주지 않는 구조를 만들 수 있다.
- 신규 유저의 행동을 일정 시간 관찰한 뒤 권한을 부여할 수 있다.

# 5. 메시지 및 임베드 기능

## 5.1 Message 기능

Zira는 기존 메시지에 역할 반응을 붙일 수도 있고, 봇이 직접 메시지를 보낼 수도 있다.

### 기본 흐름

```txt
1. 역할 선택 채널 지정
2. Zira가 메시지 생성
3. 메시지에 이모지와 역할 연결
4. 유저가 이모지를 눌러 역할 선택
```

### 메시지 예시

```txt
아래 이모지를 눌러 원하는 역할을 선택하세요.

🤖 AI 파트
💻 프론트엔드
🛠 백엔드
🎨 기획/디자인
📢 공지 알림
🏆 해커톤 알림
📚 스터디 알림
```

### 장점

- 관리자가 직접 긴 메시지를 작성하지 않아도 된다.
- 역할 선택 메시지를 봇이 일관된 형식으로 관리할 수 있다.
- 나중에 메시지를 수정하거나 역할을 추가하기 쉽다.

---

## 5.2 Embed 기능

Zira는 임베드 메시지도 생성할 수 있다.

임베드는 일반 텍스트보다 시각적으로 깔끔하기 때문에 역할 선택 페이지나 온보딩 페이지에 적합하다.

### 임베드 예시

```txt
[역할 선택]

아래 이모지를 눌러 원하는 역할을 선택하세요.

🤖 AI 파트
💻 프론트엔드
🛠 백엔드
🎨 기획/디자인

📢 공지 알림
📚 스터디 알림
🏆 해커톤 알림
```

### 장점

- 서버가 깔끔해 보인다.
- 역할 선택 화면을 보기 좋게 구성할 수 있다.
- 공지, 규칙, 온보딩 메시지에 적합하다.
- 서버의 전문성과 신뢰도를 높일 수 있다.

---

# 6. Suggestion 기능

Zira는 단순 역할 봇이지만, 제안 시스템도 제공한다.

유저가 서버 운영에 대한 제안을 제출하고, 관리자가 승인하거나 거절할 수 있다.

### 사용 예시

```txt
/suggest 다음 스터디 주제로 RAG를 다뤄주세요.
```

### 적합한 상황

- 스터디 주제 제안
- 서버 운영 개선 제안
- 이벤트 아이디어 모집
- 프로젝트 아이디어 모집
- 커뮤니티 피드백 수집
- 동아리 활동 방향 제안

### 동아리 서버 활용 예시

```txt
#suggestions

- 다음 해커톤 주제 제안
- 스터디 자료 요청
- 프로젝트 팀빌딩 아이디어
- 서버 운영 개선 의견
- 외부 행사 추천
```

### 장점

- 유저 참여도를 높일 수 있다.
- 운영진이 피드백을 체계적으로 받을 수 있다.
- 커뮤니티 운영 방향을 유저 중심으로 조정할 수 있다.

# 7. 관리 기능

## 7.1 List

**List** 기능은 현재 서버에 설정된 Reaction Role 목록을 확인하는 기능이다.

### 사용 목적

- 어떤 메시지에 어떤 역할이 연결되어 있는지 확인
- 오류가 발생했을 때 설정 점검
- 삭제할 Reaction Role 찾기
- 오래된 설정 확인
- 역할 구조 정리

### 장점

- 서버 역할 설정을 점검하기 쉽다.
- 관리자가 현재 구조를 빠르게 파악할 수 있다.
- 설정 오류를 찾는 데 유용하다.

---

## 7.2 Delete

**Delete** 기능은 특정 메시지에 연결된 Reaction Role 설정을 삭제하는 기능이다.

### 사용 상황

- 잘못 설정한 역할 반응 삭제
- 오래된 역할 선택 메시지 정리
- 더 이상 쓰지 않는 이모지 연결 제거
- 삭제된 역할과 연결된 설정 정리
- 서버 구조 개편 시 기존 설정 제거

### 주의점

Reaction Role 설정을 삭제해도 이미 유저에게 부여된 역할이 자동으로 제거되는 것은 아니다.  
기존 유저 역할은 별도로 정리해야 할 수 있다.

# 8. 권한 구조

Zira가 정상 작동하려면 Discord 서버에서 권한 설정이 매우 중요하다.

## 8.1 필요한 주요 권한

| 권한 | 용도 |
|---|---|
| Manage Roles | 역할 부여 및 제거 |
| Read Messages | 메시지 읽기 |
| Send Messages | 명령어 응답 및 메시지 생성 |
| Manage Messages | 유저 반응 제거 |
| Embed Links | 임베드 메시지 생성 |
| Read Message History | 기존 메시지 확인 |
| Add Reactions | 메시지에 반응 추가 |
| Use External Emojis | 외부 커스텀 이모지 사용 |

---

## 8.2 가장 흔한 오류

### 1. Zira 역할 위치가 낮음

Zira의 역할이 부여하려는 역할보다 아래에 있으면 역할 부여가 되지 않는다.

올바른 구조 예시:

```txt
Admin
운영진
Zira
AI Part
Frontend
Backend
Member
New Member
```

Zira 역할은 반드시 Zira가 부여할 역할보다 위에 있어야 한다.

---

### 2. Manage Roles 권한 없음

Zira에게 `Manage Roles` 권한이 없으면 역할을 부여하거나 제거할 수 없다.

---

### 3. 채널 권한 누락

Zira가 역할 선택 메시지가 있는 채널을 읽을 수 있어야 한다.

필요한 채널 권한:

```txt
Read Messages
Read Message History
Add Reactions
Send Messages
Embed Links
```

---

### 4. 커스텀 이모지 문제

외부 커스텀 이모지를 사용하려면 `Use External Emojis` 권한이 필요하다.  
또한 Zira가 해당 이모지를 사용할 수 있는 상태여야 한다.

---

# 9. 무료/프리미엄 제한

Zira는 무료로도 기본적인 Reaction Role 기능을 사용할 수 있지만, 일부 고급 기능은 제한이 있다.

| 항목 | 무료 플랜 | 프리미엄/파트너 서버 |
|---|---:|---:|
| 전체 Reaction Role | 제한 있음 | 확장 또는 무제한 |
| Toggle Role | 제한 있음 | 확장 또는 무제한 |
| Remove Role | 제한 있음 | 확장 또는 무제한 |
| Autorole | 제한 있음 | 확장 또는 무제한 |
| Timed Role | 제한 있음 | 확장 |
| Suggestion 고급 기능 | 제한 있음 | 사용 가능 |
| 우선 지원 | 제한적 | 가능 |

일반적인 소규모 동아리나 스터디 서버라면 무료 기능만으로도 충분할 가능성이 높다.  
하지만 대형 서버이거나 역할 구조가 복잡한 경우 프리미엄이 필요할 수 있다.

# 11. 장점

## 11.1 역할 자동화에 특화되어 있다

Zira는 Reaction Role 기능에 집중된 봇이기 때문에 역할 선택, 알림 구독, 인증 처리에 강하다.

## 11.2 설정 방식이 비교적 단순하다

기본 흐름은 다음과 같다.

```txt
채널 지정
↓
메시지 생성 또는 선택
↓
이모지와 역할 연결
↓
유저가 직접 역할 선택
```

## 11.3 다양한 역할 타입을 제공한다

Normal, Once, Remove, Toggle을 통해 다양한 서버 구조를 만들 수 있다.

## 11.4 임베드 메시지로 깔끔한 구성 가능

역할 선택 페이지를 보기 좋게 만들 수 있다.

## 11.5 소규모 서버에서는 무료로도 충분하다

동아리, 스터디, 게임 커뮤니티 정도 규모에서는 무료 기능만으로도 충분히 활용 가능하다.

---

# 12. 단점 및 리스크

## 12.1 역할 계층 문제에 민감하다

Zira 역할 위치가 낮으면 역할 부여가 되지 않는다.  
초기 설정 시 가장 많이 발생하는 문제다.

## 12.2 고급 기능은 프리미엄 의존도가 있다

Toggle, Remove, Autorole, Timed Role 등을 많이 사용하려면 무료 제한에 걸릴 수 있다.

## 12.3 Discord 반응 제한이 있다

한 메시지에 붙일 수 있는 반응 수에는 제한이 있다.  
역할이 많으면 메시지를 여러 개로 나누어야 한다.

## 12.4 보안 전문 봇은 아니다

Zira는 역할 자동화에는 강하지만, 다음 기능에는 특화되어 있지 않다.

- 스팸 방지
- 고급 모더레이션
- Captcha 인증
- 티켓 시스템
- 경고/밴 관리
- 로그 추적
- 레벨링 시스템

이런 기능이 필요하면 다른 봇과 함께 사용하는 것이 좋다.

# 13. Zira 사용 추천 여부

## 추천하는 경우

Zira는 다음 상황에 적합하다.

- 유저가 직접 역할을 선택하게 하고 싶을 때
- 파트별 역할을 나눠야 할 때
- 알림 구독 역할을 운영하고 싶을 때
- 규칙 동의 후 인증 역할을 주고 싶을 때
- 서버 온보딩을 자동화하고 싶을 때
- 역할 기반 채널 접근을 운영하고 싶을 때
- 관심사 기반 커뮤니티를 운영할 때

---

## 다른 봇을 고려해야 하는 경우

다음 목적이라면 Zira만으로는 부족할 수 있다.

- 강력한 모더레이션 기능이 필요할 때
- 티켓 시스템이 필요할 때
- 서버 로그 관리가 중요할 때
- Captcha 기반 인증이 필요할 때
- 레벨링/경험치 시스템이 필요할 때
- 버튼/셀렉트 메뉴 기반 최신 UI를 원할 때
- 서버 보안이 핵심일 때

# 16. 최종 평가

Zira는 Discord 서버에서 **역할 자동화와 온보딩을 설계하기에 매우 적합한 봇**이다.

특히 다음과 같은 서버에 잘 맞는다.

- 개발 동아리
- 스터디 커뮤니티
- 게임 커뮤니티
- 학교 서버
- 프로젝트 팀 서버
- 관심사 기반 커뮤니티

다만 Zira는 모든 기능을 제공하는 종합 관리 봇은 아니다.  
따라서 역할 선택과 인증에는 Zira를 사용하고, 모더레이션이나 보안 기능은 다른 봇 또는 Discord 기본 기능과 조합하는 것이 좋다.

# 17. 결론

**Zira = 역할 반응 자동화에 특화된 Discord 봇**

Zira는 서버의 역할 구조가 명확하고, 유저가 직접 역할을 선택해야 하는 커뮤니티에서 강력한 효과를 발휘한다.

특히 동아리나 스터디 서버에서는 다음과 같은 목적으로 활용하기 좋다.

- 규칙 동의 후 인증
- 파트별 역할 선택
- 알림 구독 관리
- 신규 유저 온보딩
- 서버 제안 시스템 운영
- 역할 기반 채널 접근 제어

결론적으로 Zira는 **Discord 서버 운영을 체계화하고, 관리자의 반복 작업을 줄여주는 역할 자동화 도구**라고 볼 수 있다.
